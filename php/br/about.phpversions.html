Versões do PHP documentadas neste manual

O manual contém informações sobre o passada, presente, e futuras versões do PHP. Modificações no comportamento são documentados como notas, changelogs, e em trechos nas páginas do manual. A mais antiga versão documentada é PHP 4.1.0, e a última sendo a PHP 6.x.x.

Parte da documentação existe para versão do PHP no CVS, que sempre tem as novidades. Ele já liberado online através do sistema manuseador de versão do CVS, ou via o » snapshot. Estão fique ciente que recursos marcados como "available in CVS" estão normalmente indisponíveis. Embora, vá está disponível na próxima versão estável do PHP. Para download da versão do CVS, veja a página » acesso ao CVS como anônimo.

Então, pode existir documentação para versão do PHP que não foi liberada. (Algo como PHP 6.x.x quando a mais recente versão estável é 5.x.x). Muitas vezes, isto não é erro da documentação. São explicações antecipadas para recursos ainda não disponíveis na edição atual do PHP, mas que estão previstas para essas versões do PHP. Normalmente, PHP adiciona somente novos recursos na principal versão, fora isso são somente reparo de bugs. Usando o formato de versão A.B.C, a principal versão incrementa A ou B e a menor incrementa C. Então por exemplo não é comum uma característica ser documentada como disponível no PHP x.1.x quando a última é PHP x.0.x. Também note que o manual é escrito no tempo presente, não no futuro.

Muitas vezes o manual do PHP lista "Valores Padrões" para diretivas do PHP. Estes valores são baseados no php.ini-dist e não no php.ini-recommended. Eles também referem-se a última versão do PHP. Veja o apêndice da diretiva do PHP para detalhes destes valores e mudanças.
